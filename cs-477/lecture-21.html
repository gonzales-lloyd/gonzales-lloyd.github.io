Lecture 21

Greedy algorithms address optimization problems, much like dynamic programming. Greedy algorithms generally tend to be much faster to implement and run than dynamic programming solutions, but they may not always provide optimal solutions unless the greedy choice property applies.

Often, the "correct" greedy strategy will pop out right away, but we should always try to find a counterexample to the greedy strategy.

Generally, we can think of these problems by starting from a dynamic programming perspective. For example, in our "greedy" schedule problem, our output is some set of activities. So why would we even use a greedy strategy? After all, we have a dynamic programming approach here.

  • In terms of the number of choices to consider, we have choices to make in dynamic programming. But under a greedy strategy, we only have one choice - the activity with the earliest finish time in
  • In terms of the number of subproblems in the optimal solution, we have two subproblems in the dynamic programming case; we only have one subproblem in the greedy case.

So our greedy approach is as follows:

  • Choose with the earliest finish time. This is our greedy choice.
  • Add to the set of activities used in the optimal solution.

It's worth noting that as with any optimization problem, there may be many optimal solutions. If the only thing we care about is the number of activities, it's possible the greedy and dynamic programming approaches will yield two different but equally valid answers. If there are additional criteria to optimize by, we would need to account for those as well.

So how did we get to our greedy solution?

  • Determined the optimal substructure of the problem
  • Developed a recursive solution
  • Proved that one of the optimal choices is the greedy choice
  • Showed that all but one fo the subproblems resulted by making the greedy choice are empty

To design greedy algorithms:

  • Cast the optimization problem as one for which we make a (greedy) choice and are left with only one subproblem to solve
  • Prove the greedy choice property - that there is always an optimal solution to the (generic form of the) original problem that makes the greedy choice, which we proved when demonstrating swapping with in the dynamic programming version was valid
  • Prove the optimal substructure - the greedy choice plus an optimal solution to the resulting subproblem leads to an optimal solution

The correctness of greedy algorithms not only entails giving the correct solution, but also:

  • the greedy choice property, RESOLVE
  • the optimal substructure property

In general, in dynamic programming:

  • We make a choice at each step
  • The choice depends on solutions to subproblems
  • We do a bottom-up solutoin, from smaller to larger subproblems
    But in greedy algorithm
  • We make the greedy choice and then solve the subproblem arising after the choice is made
  • The choice we make may depend on previous choices, but n ot on solutions to subproblems
  • Top-down solution, problems decrease in size
