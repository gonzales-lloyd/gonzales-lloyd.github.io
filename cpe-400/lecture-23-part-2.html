Lecture 23 part 2

Ethernet is the "dominant" wired LAN technology. It's relatively cheap to buy a network interface card (NIC) for it, and it's one of the first widely used LAN technologies. It's simple and cheap, and has so far kept up with improvements in speed.

In terms of physical topology, a bus layout was popular throughout the mid-90s. Here, a single coaxial cable was used for communication for all the hosts, which meant that all nodes were in the same collision domain. In turn, hosts could easily collide with each other.

The star topology is more prevalent today, in which a switch exists in the "middle", where each node runs a separate Ethernet protocol to the switch. Since no two hosts share the same line, there isn't a possibility of collision.

A sending adapter encapsulates an IP datagram (or some other network layer protocol packet) in an Ethernet frame.

  • The preamble consists of 7 bytes each containing 10101010, followed by 1 last byte with 10101011 instead.
  • The addresses are 6 bytes, which are simply the source/destination MAC addresses. If an adapter receives a frame with a matching destination address (or a broadcast address), it passes the data in the frame to the network layer protocol. Otherwise, the adapter discards the frame.
  • The type indicates a higher layer protocol, usually IPv4, IPv6, ARP, etc.
  • CRC is a cyclic redundancy check performed at the receiver; if an error is detected, the frame is dropped.

Something worth noting is that Ethernet is unreliable and connectionless.

  • It's connectionless in that there's no handshaking performed between sending and receiving NICs.
  • It's unreliable in that receiving NIC's don't send ACKs or nACKs to sending NICs. A stream of datagrams passd ed. The network layer can have gaps (missing datagrams), in which case it's up to a higher layer (like TCP) to wait and fill the gaps; otherwise, the app sees the gaps.

CSMA/CD for Ethernet
...

Again, remember CSMA/CD is what dictates what should happen upon a collision. The basic algorithm is as follows:

  • The NIC receives a datagram from the network layer, creating a frame.
  • If the NIC senses that the channel is idle to begin with (CSMA), it begins a frame transmission. If it senses that the channel is busy, it waits until the channel is idle before transmitting.
  • If the NIC transmits an entire frame without detecting another frame, the NIC is done with the frame.
  • However, if NIC detects another transmission while transmitting, it aborts its current transmission, sends a "jam signal", and then prepares for retransmission.
    • A jam signal is a 48-bit signal sent to all other transmitters, ensuring that they are aware of the collision.
    • After any collision, NIC enters "exponential backoff".

Exponential backoff
...

What is exponential backoff? Notice that if you send a jam signal, and the other end also sends a jam signal, you're just going to end up sending jam signals forever until every other NIC on the network can't send anything but jam signals.

The goal with exponential backoff is to adapt retransmission attempts to the current estimated load. With heavier loads, a random wait will be longer.

Common network devices
...

We have a broad variety of network devices:

  • End-hosts
  • Hubs
  • Switches
  • Bridges
  • VLANs
  • Routers
  • Firewalls

Hubs
...

Hubs are physical-layer "dumb" repeaters; bits coming in one link go out to all the other links at the same rate. All node connected to the hub can collide with one another. There's no frame buffering and no CSMA/CD as the hub itself; it's the responsibility of the host NICs to detect collisions.

Switches
...

Switches are link-layer devices that are smarter than hubs, taking an active role in ensuring network communications are successful. They store and forward Ethernet frames, examining the MAC address of incoming frames. Switches selectively forward frames to one or more outgoing links when the frame is to be forwarded on the segment; CSMA/CD is used to access the segment itself.

Switches are transparent in that hosts are unaware of the presence of switches. In general, switches are plug-and-play. They're self learning, meaning that switches typically don't need to be configured.

Switches are particularly significant in that they allow for multiple simultaneous transmissions. Since each host has a dedicated, direct connection to the switch, and the switch is able to buffer packets, the Ethernet protocol can be used on each incoming link.

Since each link is its own collision domain, and the links are full duplex, it makes switching possible. This means that A-A' and B-B' can communicate simultaneously, without collisions – something not possible with a "dumb" hub.

But how does the switch know that A' is reachable via interface 4 and B' is reachable via interface 5? It does this by maintaining a switch table, in which each entry in the table holds:

  • The MAC address of the host
  • The interface to reach the host
  • A timestamp

How is this switch table maintained and developed? The switch learns which hosts can be reached through which interfaces; whenever a frame is received, the switch "learns" the location of the sender, through an incoming LAN segment. The switch can then record the sender/location pair in the switch table.

When the frame destination is unknown, we simply flood the network by broadcasting the frame to all NICs. When the destination is known, we perform a selective send.

How are routers different? Remember that routers essentially perform the routing internal to a network; "outside" of the router is the rest of the internet.

Both switches and routers are store-and-forward:

  • The router deals with network-layer devices, examining network-layer headers.
  • The switch deals with link-layer devices, examining link-layer headers.

Also, both have forwarding tables:

  • Routers compute tables using routing algorithms and IP addresses.
  • Switches learn forwarding tables using flooding, learning, and MAC addresses.

Bridges
...

Many times, it is necessary to connect a local area network to another local area network. But the LANs might be of different types; one LAN might be an Ethernet LAN, while the other is a token ring LAN. In this case, switches aren't sufficient. (That said, a bridge can also be used to connect two similar LANs, such as two CSMA/CD LANs, in which case it functions like a switch.)

Such LAN to LAN connections are often performed with a bridge.

Just like switches, bridges don't need programming; they observe all the traffic and build routing tables from what they observe. This is known as backward learning.

Each bridge has exactly two connections (ports), with a routing table associated with each port. A bridge observes each frame that arrives at a port, extracts the source address from the frame, and then places that address in the port's routing table.

But the main benefit of bridges that's different from switches is that it can convert from one frame format to another. The bridge removes the headers and trailers from one frame format and inserts the headers and trailers for the second frame format.

VLANs
...

Suppose that a CSE (WPEB) faculty member moves their office to the EE building (SEM), but still wants to connect to the CSE switch. One option is to simply put switches everywhere. But that's inefficient.

At the same time, though, we can't just have a signale broadcast domain; all layer-2 broadcast traffic would then have to cross the entire LAN, including ARPs, DHCPs, and so on. This has security and privacy issues, not to mention efficiency issues.

So instead, we might have VLANs, which are virtual local area networks. These are switches that can be configured to define multiple virtual LANs over a single physical virtual LAN infrastructure. In port-based VLANs, the switch ports are grouped (by the switch's software) so that a single physical switch seems to operate as multiple virtual switches.

Port-based VLAN has several properties:

  • Traffic isolation: frames to/from ports 2-8 can only reach ports 2-8. You can also define a VLAN based on MAC address of the endpoints, rather than a set of switch ports.
  • Dynamic membership: Ports can be dynamically assigned among VLANs.
  • Forwarding between VLANs: this is achieved via routing, just as with separate switches. In practice, vendors tend to sell combined switches plus routers.

There are also trunk ports, which carry frames between VLANs defined over multiple physical switches. Frames forwarded within VLAN between switches can't be ordinary 802.1 frames, as they need to carry the VLAN identification information.

The 802.1q protocol adds/removes additional header fields for frames forwarded between trunk ports.
