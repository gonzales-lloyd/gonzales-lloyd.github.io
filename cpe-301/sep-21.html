Remember that from September 19, we had a system with RAM and ROM that shared the address bus like so:

  • the end of ROM is at 1111 1111 1111 1111
  • the start of ROM is at 1110 1111 1111 1111
  • the end of RAM is 0011 1111 1111 1111
  • the start of RAM is 0000 0000 0000 0000

You may notice an interesting pattern for two here:

  • For RAM, the top two bits are always 0s.
  • For ROM, all but the fourth-highest bit is always the same.

As a result, these bits are not needed.

In turn, we can actually use the top two bits for our chip select for the 16k RAM. But because they are off by default, we will need to take their complement. Therefore, if we want to select the 16k RAM, we know that it is selected if and is off.

Similarly, we can use all the bits except for for ROM. We only need one address line to address a chip. So we know that the 4k ROM is selected if all the address lines are on except for .

But clearly having 15 address lines enabled doesn't really seem to make sense. This is because we actually need at least 12 address lines to physically address all parts of ROM, so if we have 15 address lines active all the time, we can't actually address ROM. In turn, this is an invalid memory configuration.

So for example, the Arduino ATMega 2560 has 16k RAM and a UART (universal asynchronous receiver/transmitter). Both of these are mapped on the address bus using memory-mapped I/O to design a chip.

Then, we need a select for each chip when the 16k starts at 0x4000 and UART starts at 0xFC00.

We can draw a diagram to see where things lie:

UART needs 8 bytes, because it has 8 external data lines. It's not in a blocking configuration, so it only needs to be addressable through its data lines (and not directly through memory). In other words: we only need enough elements in the address space to access its data lines. We don't need to address individual parts of UART itself, as is the case with RAM, since we are only concerned with the interface.

Notice that:

  • the 16k RAM starts at 0100 0000 0000 0000
  • the 16k RAM ends at 0111 1111 1111 1111
  • the UART starts at 1111 1100 0000 0000
  • the UART ends at 1111 1100 0000 0111

Notice how the top two bits of RAM are the same, and the top 13 bits of the UART are the same.

Then, the chip select for the 16k RAM is (), while the chip select for the UART is to in a high state, and to in a low state.

Notice that we still have the 3 bits needed for the UART, and the 12 bits needed for the RAM - this is a valid memory configuration.

So the concern now is figuring out the allocation of memory.

The block address, , is the size required by the chip. is the size of the chip space. Then, the block number consists of bits, which is our chip select, and the block address consists fo bits, which is the number of addresses internal to the chip.

We only have 16 available bits, so memory-mapped I/O is not possible here. Both can be made individually with 16 bits, but we can't use them together.
