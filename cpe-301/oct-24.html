We now move to a discussion of analog-to-digital converters, or ADCs. Analog signals are continuous time signals which (generally) go on for an infinite amount of time. And of course, since there are infinitely many values in a continuous waveform (and also infinitely many time slices in an infinitely long waveform), it's not possible to represent this exactly in computers.

In computers, infinity does not exist. (Barring special representations, of course, but you still can't represent continuous signals infinitely.) So instead, we select points (usually at some fixed interval or segment), record its value, and repeat this. This is called sampling or digitization.

We sample at a predetermined rate. It's important to choose a good rate because you'll lose all the data in between each of your samples.

Suppose we have a waveform that looks like this:

If we sample it at just the green points and then graph our digitized points out, notice how we've lost some data - we don't have the same function as the original function. Furthermore, it's worth noting that although we're drawing lines to connect the samples in the green graph, there isn't actually data in between each point - we've interpolated, or "added", that data based on the data we actually have.

The bandwidth of a signal is the range of frequencies found within that signal. Recall that a frequency is the number of repeatable occurrences over one unit of time.

Suppose that we have the following waveform:

There are four different frequencies in this signal, but we may not want to sample all of them. Something important to note is that any complex signal can be represented as a combination of sine waves for each frequency within the bandwidth. This is known as the Fourier theorem.

The Fourier theorem states that any continuous periodic waveform can be represented by a potentially infinite series of sines and cosines. (Recall that cosines are just phase shifted sines, which is why we only said "sines" above.)

This is quite important in the fact that we can recreate complete signals using only sampled information. Of course, you have to sample it enough for this to work - the Nyquist-Shannon theorem tells us that we need to sample it at least twice as fast as the frequency of the densest (or highest frequency) part of the signal.

So, if we want to work with a signal in a computer, it goes through a certain process "chain":

  • It runs through a low pass filter
  • It passes through the sample and hold, which samples the value and holds it
  • We pass these samples through the ADC
  • Those are passed to some form of storage
  • The stored digital "signal" is passed to a DAC
  • The DAC passes this through a a reconstruction filter
  • We get the original function back out

The low pass filter attempts to remove frequencies above the Nyquist limit, since the Nyquist-Shannon theorem tells us that we don't need these frequencies to reconstruct the signal. It doesn't always work, though; sharper curves cause more phase distortion.

The sample and hold, an analog circuit also known as S/H, deals with the fact that incoming signals can vary in amplitude. If the signal changes in the time that the ADC is processing the previous value - that is, the signal changes early- the ADC wlil yield improper readings. Of course, sample and holds can only hold a finite number of values; once this is full, we will start getting improper readings.

The ADC quantizes samples, approximating them to some prescribed finite set of values. Each sample is converted to binary, with the sampling rate determining the highest frequency that can be represented by digital samples.

The number of bits in quantization dictates the signal-to-noise ratio. The more bits you have to work with (in general), the more accurate you will be; if you can represent a signal with 256 values, that's more accurate than 8 values.
