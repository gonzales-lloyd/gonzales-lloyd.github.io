Chapter 4

In general, the Intel/AMD family of processors are used for servers, desktops, and laptops. They're CISC (complex instruction set computers), and are more complicated than most other CPUs and are not ideally suited for a first course in computer organization.

There's also MIPS CPUs (R3000, R4000), which are widely studied in computer organization courses because of a widely-used textbook called Computer Organization and Design. However, the architecture is over 30 years old, not widely used commercially, and the textbook is built for a senior-level class.

RISC-V is a newer CPU family, introduced in 2010. The intellectual property is free and open source, but does not yet have the same breadth and quality of tools that are available for other popular CPUs. It is not yet clear whether or not RISC-V will be used outside academia. RISC-V is a foundation that defines the processor - they don't actually build the hardware, unlike Intel/AMD.

Another family is AVR, which is an inexpensive 8-bit processor used in Arduino boards. It was formerly developed by Atmel, and is now owned by Microchip.

This class studies ARM, which are similar in complexity to other CPUs used for learning purposes. It's well-documented with many resources for learning it. Raspberry Pis use ARM CPUs, and have been a commercial success especially in the mobile market. One major reason is that ARM offers a low-power solution, which was important as mobile competing took off. Unlike Intel and RISC-V, ARM sells the IP of the CPU rather than the IC itself.

Of course, commercial success does not imply technical superiority. At the machine language level, processors tend to have a lot in common, and in this class, we'll use ARMv7 because of the tools and documentation available to it. In addition, this class only covers the instructions common in other processor families.

What's in a computer?
...

There are (at an extremely high level) five levels to a computer - input interfaces (like a mouse, keyboard, or camera), output (like a display or hard disk, which can be both input and output), the control part of the CPU, the data path part of the CPU, and memory.

We say that a microprocessor is a CPU on an integrated circuit. We can loosely define a microprocessor as the "brains" of a computer, and is the most important IC in a computer.

More specifically, we can think of a microprocessor as a digital circuit (a state machine) that accepts data as input, processes it according to instructions stored in memory, and provides results as output.

But what is the difference between a program and data? A program is like a recipe, and the data is the ingredients. "Data is what a computer knows; programs are what a computer does."

Programs and data are stored in a computer's memory. (Since we almost always use Von Neumann architectures here, we can reasonably assume that both are stored in memory.) The processor fetches program instructions from memory and executes them. Data can aulos come from input devices.

Now, the first microprocessors were relatively large ICs. The first ARM (Advanced RISC Machine) chip in 1985 was 7x7 millimeters, and had 6,000 gates. Computer systems used to have a lot of ICs on a printed circuit board each with their own function, but the shrinkage of circuit components led to the merger of various memory and peripheral components.

The term microcomputer, or system on a chip, can be used to describe a microprocessor integrated with memory and peripherals on a single IC. Sometimes, the term "microcomputer" may be used to mean microprocessor. (Below is a microcomputer, with the microprocessor towards the top connected to peripherals and other components.)

A general purpose CPU can implement a complete range of computing tasks, meaning that it is fully programmable. The theoretical definition was described by Alan Turing, and a programming language or general purpose CPU can be described as Turing complete. A Turing complete system can be proven mathematically to be capable of performing any possible calculation.

Note that Turing completeness is not related to the Turing test.

Over time, we developed domain-specific architectures, rather than general-purpose CPUs. For example, a graphics processing unit, or GPU, was optimized towards performing the calculations typical of graphics rendering. Other architectures are network computers and TensorFlow.

In general purpose computing, there are still many variations. For example, desktop, server, and embedded competing are all different areas. A PC is not the same thing as a web server and is not the same thing as a microwave; all of these have different purposes with different needs.

The processor
...

The number of data bits a CPU can access in one cycle is a key metric. Our CPU is 32 bits, and is therefore a 32-bit processor. Other common sizes are 8, 16, 64 bits, or more. The data bus is bidirectional, and the CPU can read or write across the data bus.

Observe that the image above shows that it has 32 bits of addressable memory. Note that this is not necessarily how we name the bits to a processor; it's the number of data bits a CPU can access in one cycle. It may be the case that the address bus is less than 32 bits.

Memory typically contains non-volatile memory, like flash memory, to hold programs. It also contains volatile memory in the form of RAM to hold data.

Now, if we have a 32-bit wide address, this supports 4GB of memory. You might think that this means we actually can support 16 GB of memory because each address holds 32 bits of memory, but this is not true. In reality, each address only refers to 1 byte of memory.

But we usually want to access data 4 bytes at a time, as shown above. As a result, addresses are usually aligned to 32-bit words, rather than an individual byte.

Now, there are two ways to pack bytes into words. You can express it in little endian, where the least significant byte comes last, or big endian, where the most significant byte comes last. ARM processors can be programmed to do it either way, but default to little endian.

For larger processors, the address bus is generally the same size as the data bus - a 32-bit processor has 32-bit addresses and data buss. However, for 8-bit processors, the address bus is generally larger than 8 bits.

To execute an instruction, the processor must first fetch it. To fetch an instruction, the processor outputs the address of the instruction. Then, the memory outputs the instruction on the data bus. The processor stores it internally, and then executes it.

In our CPU, every machine-language instruction is a 32-bit number. (Instruction sizes may vary from processor to processor.) We can fetch one instruction each cycle. In comparison, a 64-bit ARM CPU still has 32-bit instructions; as a result, the CPU can fetch two instructions per cycle.

Instruction set architecture
...

A processor's machine language and assembly language are part of its instruction set architecture, or ISA. This includes:

  • The processor state visible to the program, including registers, flags, status bits, and control bits
  • The instruction set and data structures
  • Data types, their size, and their representation
  • The system model (states, privileges, and exceptions)
  • The memory model

The computer architecture is the programmer's model of the CPU, a "contract" between the CPU designers and the programmer or compiler, defining behavior.

ISAs are important historically because computer manufacturers could make a family of computers that used a common ISA. Before 1964, every computer had an instruction set architecture that was unique to that computer. Making a common ISA eliminated the need for programmers to keep learning how to make different computers. This was the beginning of hardware compatibility, making it easier to upgrade from one computer to another, and gave IBM a large advantage.

Each ISA has a family of procssrrs that it supports. The latest is ARMv8, which is in new Raspberry Pis. ARM succeededed because they could be used across a wide range of hardware devices, through their common platform of code and tools. A very large range of CPUs with varying power could use the same instruction set.
