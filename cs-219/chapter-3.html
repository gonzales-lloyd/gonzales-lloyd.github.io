Analog vs. digital
...

You can have two types of signals - analog and digital. There are infinite possible values on an analog signal, but finite on digital. In binary, there are only two possible values.

Both analog and digital signals can lose quality, due to noise or other reasons. A digitzed version is much more resilient against noise - although the voltage level may still not be kept perfectly, sampling at points when the voltage has stabilized yields better results.

A switch has a source input, a control input (which controls whether that current can flow), and an output. Switches were originally relays, eventually moving onto vaccum tubes, discrete transistors, and eventually integrated circuits. It is possible to make a computer using analog circuits, which was the case for the Norden bombsight. However, digital computers won because:

  • digital signals have greater noise immunity, as described above
  • it is easier to make transistors (the basic digital electronic component) behave like switches
  • digital computers are easily reprogrammable (and also easier to debug) - this is hard in an analog circuit, which generally must be made for one single, specific purpose
  • it is easier to get high-resolution values in a digital circuit (you can use 32 bits to get a number within a resolution of a few billion; trying to get the same precision on analog would be very difficult).

Logic and bitwise operations
...

George Boole wrote The Laws of Thought, explaining what came to be known as Boolean logic, the basis for digital circuit design. Another major person was Claude Shannon, who in 1937 laid down the foundations of digital circuit design by using switches that could implement Boolean logic.

Transistors are the basic building blocks of all circuits; an input into a MOSFET (metal–oxide–semiconductor field-effect transistor) either blocks or permits the flow of electricity. (See slide 16.)

Logic gates exist. You can express the state of a digital circuit's inputs and outputs over time using a timing waveform. Timing waveforms are typically idealized, since transitions have a nonzero propagation delay.

There are also multiple input logic gates. The notable multi-input gate is a multi-input XOR, which is true if an odd number of its inputs is true.

You can use this on a string of bits. For example, if you want to select only a few bits in a number, you could use a mask with an AND. If you wanted to mask the bottom half of an 8-bit number, use 00001111 with a bitwise AND to isolate the bottom 8 bits.

You can use bitwise operations in C - &, |, ^, and ~ are bitwise AND, OR, XOR, and NOT, accordingly. (This is different from the logical && and ||. A "logical XOR" can be achieved by checking for inequality (!=).

You can also do logical and arithmetic shifts in C/C++.

uint32_t x;
int32_t y;

x = x << 1; // logical shift left
x = x >> 1; // logical shift right
y = y >> 1; // arithmetic shift right (automatically detected if signed, but is implementation-dependent)

Combinational circuits
...

To implement a digital binary adder, you use half adders and combine them to make full adders. In a half adder, you put in two inputs, and get two inputs out - the sum and the carry out. The sum is a XOR b, and the carry out is a AND b.

A full adder takes two bits and a carry in, and generates a sum and carry. In one circuit, it ANDS all the combinations of a, b, and the carry in, and ORs the result of that to generate the sum. The carry out is a 3-input XOR. Chaining many full adders together allows you to create an n-bit adder.

You can also create an n-bit subtractor by inverting the subtrahend (in a-b, it's b) to take its one's complement before passing it through the full-adders, and then adding a carry-in of 1 to the least significant full adder.

You could also create an n-bit adder and subtractor with a setting bit, where the subtrahend is passed through a XOR gate which is also connected to to the setting bit, inverting B if the setting bit is equal to 1. It is also connected to the input of the least significant full adder.

To simplify drawing digital circuits, we usually bundle or group related signals into a bus; for example, an input marked A[3:0] is a four-bit input with A[0] as the LSB and A[3] as the MSB.

We can combine many operations into one circuit, called an arithmetic and logic unit (ALU), taking two arbitrary (usually 8) n-bit inputs. It has 8 distinct functions - AND, OR, XOR, NOT, addition, subtraction, return A, and return B. (See slide 36.)

Multiplexers select a specific input from 2^n inputs from n select bits.

Combinational logic is where the output is a function of the current inputs; when an input changes, the output changes immediately, neglecting any propagation delay. In a sequential digital circuit, the output can depend on previous values of the input. Sequential circuits can store previous values of the input.

A decoder outputs a high based on n inputs, with 2^n outputs. You can also put an enable bit on this, too.

Things that are indicative of combinational circuits are

  • Gates (and, xor, or, not)
  • Adders, subtractors
  • Multiplexers
  • Decoders
  • ALU
    Things that are indicative of sequential circuits are
  • Flip-flops
  • Registers
  • Register files
  • RAM
  • State machines

Register files
...

There's the D flip-flop, which changes to the value currently on the D data bit on the clock rise (positive/rising edge). A register is just an n-bit D flip-flop, which is essnetially just a bunch of internal D flip-flops connected to each individual input bit. There may also be an enable bit.

Now, you may have an (arbitrary width) decoder and multiplexer connected to multiple registers in what's known as a register file. For example, the decoder takes in the target register (specified as WR_ADDR), and then outputs the data (of arbitrary width) in Wr_data to the target register.

Likewise, you may also have an extended-purpose multiplexer, which takes in RD_ADDR to specify the target register to read from, and then spits out the data currently contained in the register.

The WR option determines whether to write (1, or HIGH) or read (0, or LOW) on each clock cycle.

RAM
...


RAM is not directly connected to the main clock; instead, write data is stored on the rising edge of nSel.

At any given moment, you can either read or write one memory location on RAM at a time, but not both. Memory locations are random access, and can be read or written in any order; they don't have to be accessed sequentially.

Similar to a register file, R_nW ("read not write, sometimes R/W") controls whether RAM reads or writes. If R_nW is 1, then it reads; if R_nW is 0, it writes.

The data bus is bi-directional; for a write, the interface (whatever is plugged into RAM) drives the data bus. For a read, the RAM drives the data bus. Otherwise, it is Hi-Z (look this up).

A RAM with 16 address bits and 8 data bits is a 64K x 8 RAM.

State machines
...

The next state of a state machine is a function of the current state and its current inputs. This is particularly important because CPUs are state machines. For example, a vending machine needs to store whether or not to dispense something when a button is pressed; dispensing is dependent on the money previously provided to the computer.
