Chapter 6

Business decisions are only as good as the quality of the information actually used to make the decisions. You never want to find yourself using technology to help you make a bad decision faster.

This brings us to the idea of information quality. What are some characteristics of high-quality information?

  • Accuracy: Are all the values correct? Are names spelled correctly? Is the dollar amount recorded properly?
  • Completeness: Are any of the values missing? Is the address complete including street, city, state and zip code? If I have your phone number, do I have the area code as well?
  • Consistency: Is aggregate or summary information in agreement with detailed information? Do any "total" fields actually equal the true total of the individual fields?
  • Uniqueness: Is each transaction, entity, and event represented only once in the information? Are there any duplicate customers?
  • Timeliness: Is the information current with respect to the business requirements (real-time, by the second, by the minute, etc.)? Is information updated weekly, daily, or hourly?

Here's an example of low quality information:

But where does low quality information come from? The four primary sources of low quality information include:

  • Customers intentionally enter inaccurate information to protect their privacy.
  • Different entry standards and formats, where you might get all kinds of nonsensical information typed into a field – maybe you have a numeric entry form that accepts phone numbers, but also a numeric entry form that accepts any old number, and they both map to the same database
  • Operators enter abbreviated or erroneous information by accident or to save time
  • Third-party and external information can contain inconsistencies, inaccuracies, and errors

But why bother getting good information?

  • High quality information can significantly improve the chances of making a good decision.
  • Good decisions can directly impact an organization's bottom line.

Information is everywhere in an organization. Normally, information is primarily stored in relational databases. A database is a container that maintains information about various types of objects (inventory), events (transactions), people (employees), and places (warehouses). You can store whatever you want in that database, and (most of the time) you'll have a program that serves as an easier interface.

The database is the container – the programs are connected to that container. But what is a relational database? Usually, instead of just having one massive table, you often split up the table into several tables that records each logical "type" differently – you might have a table for products, customers, and orders separately, instead of just one big table for every single order.

Each table has connections with each other. While it can be hard to look at a database using just queries and pure tables instead, you can instead use some form of an interface. A UML diagram is often used to simplify the actual connections and fields of the database, while a program helps you actually see the contents of the database.

How do we talk about databases? We often store data elements in entities and their attributes:

  • An entity is a person, place, thing, transaction, or event about which information is stored – it's also known as a table, or a collection of rows.
    • An entity is not a specific instance; you might consider a collection of class instances instead.
  • An attribute (also known as fields or columns) are the data elements associated with an entity. The columns in each table represent and contain the attributes.
  • A record is a collection of related data elements – it's one row, and represents a single specific class instance.

An entity-relationship diagram, or ERD, is a flowchart that illustrates how tables in a database will relate to each other. It could look like this:

Something that's a little different is a database management system, which is software that allows users to create, read, update, and delete data in a relational database. Microsoft Access is one such example – the underlying database is the Access Database Engine, formerly the Microsoft Jet Database Engine.

Here's an example of what a relational database might look like:

Primary keys and foreign keys identify each of the various entities (tables) in the database.

  • A primary key is a field, or group of fields, that uniquely identifies a given entity in a table. These are things like Social Security numbers.
  • A foreign key is a primary key of one table that appears as an attribute in another table. It can be used to provide a logical relationship among the two tables.

But what's the point of a relational database for a business? Some advantages from a business perspective include:

  • Increased flexibility
  • Increased scalability and performance
  • Reduced information redundancy
  • Increased information integrity (quality)
  • Increased information security

This is why companies might prefer a database over a Microsoft Excel file, for example. Let's talk about each of these advantages.

Increased flexibility
...

A well-designed database should:

  • Handle changes quickly and easily. If I later want to add the employee who created a given order, I should be able to do that (and make conclusions from that new field).
  • Provide users with different views.
  • Have only one physical view – that is, if you go to the server, you know that the physical database only has one view. This deals with the physical storage of information on a storage device.
  • Have multiple logical views – that is, this focuses on how individual users logically access information to meet their own particular business needs. Not everybody needs access to the entire employee database; you want some people to have full access, while others may not. This allows for views of select tables and relationships.
    • A single view, for example, might only allow a user to see 3 out of the various tables in the database, and see the relationships and entries within each of them.

Increased scalability and performance
...

A database must scale to meet increased demand while still maintaining acceptable performance levels.

  • Scalability refers to how well a system can adapt to increased demands. If I have 100 customers today, I should be able to increase the number of customers without compromising the database's performance.
  • Performance refers to how quickly a system can perform a certain process or transaction.

Reduced data redundancy
...

Data redundancy is the duplication of data or storing the same information in multiple places. If I have the same customer, I would rather know that the string "Mary" refers to the same "Mary" everywhere, with all the same information, instead of having ten different entires that all contain the same thing.

Increase information integrity
...

Information integrity measures the quality of information.

An integrity constraint is a rule that help ensures the quality of information. While it's true we can't force users to give "real" data, we can at least force users to always give a phone number when we ask them for a phone number, preventing inserting strings and names. There are also some other forms of constraints:

  • A relational integrity constraint is a rule that enforces basic and fundamental information-based constraints. One such constraint might be preventing order creation for customers that don't exist.
  • A business-critical integrity constraint is a rule that enfrnces business rules vital to an organization's success and often require more insight and knowledge that relational integrity constraints, like a grocery store dairy return. For example, maybe we might put a rule in place that prevents users from returning milk anytime after 5 minutes after purchase.

Increased information security
...

While you can hide rows and enable password protection in Excel, it's not really meant for security. But on the other hand, databases offer several security features:

  • A password – provides user authentication
  • Access levels – determines who has access to the different types of information
  • Access control – determines types of user access, such as read-only access

For example, a CEO might be given an access level that allows them to access every single database. But since they're not really in the business of data entry, they might only have read-only access. On the other hand, an accountant might be given the power to write to the database, but only tables relevant to the their role.

Business intelligence
...

This is great, but we've also talked about using multiple sources of data to form business intelligence. A database just stores data.

A business intelligence system is often connected to a data warehouse, which is not the same thing as a database. A data warehouse is a logical collection of information gathered from many different operational databases that supports business analysis activities and decision-making tasks.

The important thing (bolded in the slides) – the primary propose of a data warehouse is to aggregate information throughout an organization into a single repository for decision-making purposes.

For example, the WCSD has databases on all the staff. But there's also Infinite Campus, which keeps track of all the students and their grades, as well as ACT and SAT test score systems. These didn't talk to each other. But if we connect them all up to a data warehouse, we can get business intelligence. Resolve: 12:56 pm

ETL, or extraction, transformation, and loading, s a process that extracts information from internal and external databases, transformation the information using a common set of enterprise definitions, and loads the information into a data warehouse.

Multidimensional analysis
...

Databases contain information in a series of two-dimensional tables, which are column and rows.

In a data warehouse or a data mart, information and multidimensional, since it contains several layers of databases – layers of columns and rows – which are referred to as cubes, a common term for the representation of multidimensional information. Here's what this might look like:

You can answer questions about not only specific promotions, stores, and products, but also specific slices of one or more of these attributes. This allows you to ask and answer high-level questions very quickly. It's true – you could just manually go to each separated system and get the data you need and clean it up yourself. But a business intelligence system connected to a data warehouse can do it much more quickly (and possibly more correctly) than you can.

Information cleansing or scrubbing
...

An organization must maintain high-quality data in its data warehouse. Information cleansing or scrubbing is a process that weeds out and fixes or discards inconsistent, incorrect, or incomplete information. This is particularly true when you get data from other sources – you can improve the quality of your own databases through your own systems, but that doesn't mean you'll catch everything (and it also certainly doesn't apply to external sources).
